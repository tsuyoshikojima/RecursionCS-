# プロセス

プログラムとはコンピュータに実行させる処理の順番のことです。
一方、プロセスとはプログラムがメモリーにロードされ実行中のものを指します。

### プロセス

各プロセスは独自のメモリ空間を持っています。

- コード領域
- データ領域
- スタック領域

また、カーネルの内部では各プロセスを管理するために`プロセス制御ブロック`(process control block)と呼ばれるものに
各プロセスに関する情報を格納している。

### プロセス制御ブロック

プロセス制御ブロックには以下の情報が含まれる。

- プロセス管理情報(プロセスそのものを管理するための情報)

    - レジスタの退避領域(プロセス切り替えの時にレジスタを退避させるための領域)
    - プロセスの属性(プロセス識別子やプロセスの所有者など、プロセスに関する雑多な付加情報)
    - 統計情報(プロセスが動いた時間、使用しているメモリ量など)

- メモリ管理情報(各メモリ領域がどのアドレスから始まるのか、その大きさは何バイトかといった情報)

- ファイル管理情報(ファイル操作に関する様々な情報)

### 具体的なプロセス確認

`ps -a`コマンドでプロセス識別子を確認する。

Linuxシステムでは、プロセスに関する情報は`/proc/{pid}`ディレクトリで確認できる。
通常、ファイルはディスク上に保存されているが、`/proc`は実態をディスク上に持たない仮想ファイルシステムで
カーネルが動的に提供している。

`cat /proc/{pid}/status`コマンドでプロセスに関する情報を確認することができる。
